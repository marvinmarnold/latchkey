Phase 2 design summary

  • No pre-deposit — caller does one usdc.approve(LatchkeyBilling, large_amount), then uses the proxy immediately
  • Off-chain accounting — wallet_state.accrued_usd tracks debt in dollars (canonical unit); conversion to USDC atomic units only at pull time
  • Crash-safe pull — sign tx locally → persist (snapshot + hash + raw) → broadcast. On restart: re-broadcast the saved raw tx (same nonce+payload = same hash = idempotent)
  • Hot path is RPC-free — blocked check + known-wallet bypass; first-seen wallets do one allowance read then cache
  • Activation gate — all Phase 2 behaviour keyed on BILLING_CONTRACT_ADDRESS; empty = Phase 1 mock (prod is safe until deploy)
